Web Security

How would you advise a potential client to secure his/her website from web security attacks? Note: the word count for your answer should not be greater than 250 words

To make sure my client's website is secure i would advise to:

- Never trust data from the browser, always sanitize data and always assume the worst

- To only show the last four digits of potential creditcards, only store and display needed data

- Implement a automatic timeout if the user has been inactive after a period of time, normally this is made by backend developers, but as frontend developers we have to take this into consideration with the API.

- Sanatising all datapackages to avoid security threats and SQL injection, this can be done manually or via a trusted Third Party Solution

- Requiring a long user password which includes numbers, small and capital letters. Aditionally you can add a two-factor authenticator.

- Use a CI/CD tool which will automatically look through any new changes and flag anything that doesnt pass the checks it goes through

- Configure the web browser to use both HTTPS and HSTS

- Take use of Vulnerability scanning tools
