Web Security

How would you advise a potential client to secure his/her website from web security attacks? Note: the word count for your answer should not be greater than 250 words

To make sure my client's website is secure i would advise to:

- To only show the last four digits of potential creditcards

- Implement a automatic timeout if the user had been inactive after a period of time, normally this is made by backend developers, but as frontend developers we have to take this into consideration with the API.

- Sanatising all datapackages to avoid security threats and SQL injection, this can be done manually or via a trusted third party Solution

- Requiring a long user password which includes numbers, small and capital letters. Aditionally you can add a authenticator.

- Use a CI/CD tool which will automatically look through any new changes and flag anything that doesnt pass the checks it goes through
